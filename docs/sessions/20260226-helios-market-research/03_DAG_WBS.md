# DAG and Work Breakdown Structure

## Execution DAG
- Phase 0 foundation and bootstrap deliverables stabilize terminal runtime, PTY lifecycle, and control-plane baseline.
- Phase 1 extends protocol, policy, provider, and persistence foundations needed by enforcement paths.
- Phase 2 implements switching, gating, governance, and integration-hardening behavior over the core runtime.
- Phase 3 hardens durability, safety, and end-to-end reliability across session replay and compliance signals.
- Phase 4 completes parity, release readiness, and final hardening gates before rollout.
